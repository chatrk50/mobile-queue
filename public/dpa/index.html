สัญญาการประมวลผลข้อมูลส่วนบุคคล PDPA-compliant

Data Processing Agreement (DPA) ตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 มาตรา 40–41 · เวอร์ชัน 1.0 · 2026

วิธีใช้: สัญญานี้เป็นแม่แบบสำหรับเจ้าของร้านที่ใช้ MobileQueue — กรอกชื่อร้าน วันที่ และชื่อผู้มีอำนาจลงนาม จากนั้นพิมพ์ (Ctrl+P) หรือบันทึกเป็น PDF เก็บไว้เป็นหลักฐาน · สัญญานี้ใช้คู่กับ ข้อกำหนดการใช้บริการ และ นโยบายความเป็นส่วนตัว

1. คำนิยาม

  • "ข้อมูลส่วนบุคคล" — ข้อมูลที่ระบุตัวตนลูกค้าได้ เช่น ชื่อ เบอร์โทร LINE user id ประวัติออเดอร์ คะแนนสะสม
  • "ผู้ควบคุมข้อมูล" — เจ้าของร้านผู้กำหนดวัตถุประสงค์และวิธีการประมวลผล
  • "ผู้ประมวลผลข้อมูล" — MobileQueue ผู้ประมวลผลตามคำสั่งของผู้ควบคุมเท่านั้น
  • "บริการ" — ระบบ POS · คิวสั่งซื้อ · สะสมแต้ม · LINE Order · รายงานและวิเคราะห์ยอดขาย

2. วัตถุประสงค์และขอบเขตการประมวลผล

ผู้ประมวลผลดำเนินการประมวลผลข้อมูลส่วนบุคคลเฉพาะเพื่อ:

  • ให้บริการตามที่ผู้ควบคุมสั่ง (รับออเดอร์ แจ้งคิว สะสมแต้ม รายงานยอดขาย)
  • ปรับปรุงความเสถียรและความปลอดภัยของบริการ
  • ปฏิบัติตามกฎหมายที่ใช้บังคับ

ผู้ประมวลผลจะไม่นำข้อมูลไปใช้เพื่อวัตถุประสงค์อื่นโดยไม่ได้รับอนุญาตเป็นลายลักษณ์อักษร

3. ประเภทข้อมูลและเจ้าของข้อมูล

  • ข้อมูลลูกค้าของร้าน: ชื่อ เบอร์โทร LINE user id รายการสั่งซื้อ คะแนนสะสม ระดับสมาชิก
  • ข้อมูลการทำธุรกรรม: วันเวลา จำนวนเงิน วิธีชำระ รายการสินค้า
  • ข้อมูลพนักงาน: ชื่อ บทบาท บันทึกการเข้าระบบ (เพื่อความปลอดภัย)

4. หน้าที่ของผู้ประมวลผล

  • ประมวลผลตามคำสั่งของผู้ควบคุมเท่านั้น ไม่เกินขอบเขตที่กำหนด
  • รักษาความลับและความปลอดภัยของข้อมูล (การเข้ารหัส การแบ่งแยกผู้เช่า การควบคุมการเข้าถึง)
  • แจ้งผู้ควบคุมโดยไม่ชักช้าเมื่อพบเหตุละเมิดข้อมูลส่วนบุคคล (ภายใน 72 ชั่วโมงตามความเป็นไปได้)
  • ลบหรือส่งคืนข้อมูลเมื่อสัญญาสิ้นสุดตามที่ผู้ควบคุมร้องขอ
  • ให้ความช่วยเหลือผู้ควบคุมในการตอบสนองต่อการใช้สิทธิของเจ้าของข้อมูล (ส่งออก/ลบ)
  • ไม่เปิดเผยข้อมูลแก่บุคคลที่สามโดยไม่ได้รับอนุญาต ยกเว้นตามที่กฎหมายกำหนด

5. หน้าที่ของผู้ควบคุมข้อมูล

  • แจ้งลูกค้าเกี่ยวกับการเก็บรวบรวมและการใช้ข้อมูล (นโยบายความเป็นส่วนตัวของร้าน)
  • ขอความยินยอมจากลูกค้าก่อนเก็บข้อมูล หรือมีฐานทางกฎหมายอื่นที่เหมาะสม
  • จัดการคำร้องสิทธิของลูกค้า (ใช้ฟีเจอร์ "ส่งออก" และ "ลบข้อมูลลูกค้า" ในระบบ)
  • แจ้งผู้ประมวลผลเมื่อต้องการลบข้อมูลหรือยกเลิกบัญชี

6. ผู้ประมวลผลย่อย (Sub-processors)

ผู้ประมวลผลใช้ผู้ให้บริการภายนอกดังต่อไปนี้ ภายใต้ข้อกำหนดการประมวลผลที่เทียบเท่า:

  • Turso / libSQL — ฐานข้อมูล (เซิร์ฟเวอร์ภายใน EU/US)
  • Render — โฮสติ้งแอปพลิเคชัน
  • SendGrid (Twilio) — ส่งอีเมลแจ้งเตือน
  • Omise (Opn Payments) — ประมวลผลการชำระค่าบริการ (ไม่เก็บเลขบัตรในระบบของเรา)

ผู้ประมวลผลจะแจ้งผู้ควบคุมเมื่อมีการเปลี่ยนแปลงผู้ประมวลผลย่อยที่มีสาระสำคัญ

7. การรักษาความปลอดภัย

  • การเข้ารหัส: รหัสผ่านใช้ scrypt · การส่งข้อมูลผ่าน TLS 1.2+
  • การแบ่งแยกข้อมูล: ข้อมูลแต่ละร้านถูกแยกด้วย tenant_id ในทุก query
  • การควบคุมการเข้าถึง: RBAC (เจ้าของ/ผู้จัดการ/แคชเชียร์) + PIN hash + session ลงนาม HMAC
  • Security headers: HSTS · CSP · X-Frame-Options · nosniff
  • การตรวจสอบ: บันทึก audit log ทุกการกระทำสำคัญ

8. การโอนข้อมูลไปต่างประเทศ

ข้อมูลอาจถูกประมวลผลในเซิร์ฟเวอร์นอกประเทศไทย (EU/US) ผ่านผู้ให้บริการที่ได้มาตรฐาน GDPR/ISO 27001 ซึ่งถือว่ามีมาตรการคุ้มครองที่เพียงพอตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคลฯ มาตรา 29

9. ระยะเวลาและการสิ้นสุดสัญญา

สัญญานี้มีผลตราบเท่าที่ผู้ควบคุมใช้บริการ MobileQueue · เมื่อสัญญาสิ้นสุด (ยกเลิกบัญชีหรือหมดอายุ) ผู้ประมวลผลจะลบข้อมูลภายใน 30 วัน นับจากวันที่สัญญาสิ้นสุด เว้นแต่มีข้อกำหนดทางกฎหมายให้เก็บรักษาต่อ · ผู้ควบคุมสามารถส่งออกข้อมูลด้วยตนเองก่อนปิดบัญชีผ่านปุ่ม "ส่งออก (JSON)" ในหน้าตั้งค่า

10. ข้อพิพาทและกฎหมายที่ใช้บังคับ

สัญญานี้อยู่ภายใต้กฎหมายแห่งราชอาณาจักรไทย · ข้อพิพาทให้ระงับโดยการเจรจา หากไม่สำเร็จให้ยื่นต่อ ศาลที่มีเขตอำนาจในประเทศไทย
