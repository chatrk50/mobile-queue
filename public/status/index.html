🚦 สถานะระบบตอนนี้

กำลังตรวจสอบ…
ตัวทดสอบ (UAT) ใช้งานได้ทดลองได้เต็มที่ ข้อมูลไม่กระทบร้านจริง · รีเซ็ตเองทุกครั้งที่รีสตาร์ท
ร้านจริง (Production) ยังเป็นเวอร์ชันเดิมของใหม่ทั้งหมดยังไม่ขึ้นร้านจริง — รอเจ้าของสั่ง “ขึ้น prod” เมื่อทดสอบพอใจ
แคชเชียร์ หน้าขาย/จัดการ หน้าลูกค้า สั่ง/จ่าย/คิว จอแสดงคิว หน้าร้าน

✅ ทำเสร็จแล้ว (ขึ้น UAT)

  • ระบบจ่ายก่อนรับคิว — แจกหมายเลขคิวหลังชำระเงินเท่านั้น
  • พร้อมเพย์ 2 แบบ — ธนาคารอื่น (QR ไดนามิก) / KBank (กรอกยอดเอง)
  • จ่ายที่เคาน์เตอร์ — ออเดอร์อ้างอิงเลขที่ กันจ่ายซ้ำ
  • หน้าจัดการแบบเลือกซ้าย-แก้ขวา — เมนู · สาขา · สต๊อก · พนักงาน (รูปแบบเดียวกันหมด)
  • ราคา Delivery แยกจากหน้าร้าน — Grab / LINE MAN / Shopee ใช้ราคาเดลิเวอรี่
  • สต๊อก/วัตถุดิบ + สูตร (BOM) — ตัดสต๊อกอัตโนมัติเมื่อขาย · เตือนของใกล้หมด · เมนูที่ทำไม่ได้ขึ้น “หมด”
  • เวลาเปิด-ปิดรายสาขา — ปิดรับออเดอร์อัตโนมัตินอกเวลา
  • รายงาน — ยอดขาย/คืน/ของเสียแยกกัน · ความพึงพอใจ · ลูกค้าซื้อซ้ำ · กระทบยอดแยกช่องทาง
  • คิวอัปเดตสด — ออเดอร์เด้งเข้าจอแคชเชียร์ทันที (heartbeat + กันหลุด)
  • แบรนด์ — ฟอนต์ Space Grotesk/IBM Plex Sans Thai · สี Arctic Paradise Blue · คีย์แพด PIN

✅ ทำเสร็จแล้ว (รอบใหม่ · ขายดี KhaiDee SaaS)

  • ออเดอร์ค้างชำระหายเอง — auto-void อัตโนมัติ ตั้งเวลาได้ใน ⚙ ตั้งค่าระบบ
  • ไอคอน SVG ชุดเดียว — ระบบแคชเชียร์ใช้ SVG line icons ครบแล้ว
  • ภาพรวม/กราฟ KPI — ยอดขาย · เมนูขายดี · ช่วงเวลาพีค (ปุ่ม "ภาพรวม/กราฟ")
  • แพลตฟอร์ม SaaS (ขายดี KhaiDee) — multi-tenant · billing (Omise) · dunning · ทดลอง 60 วัน · referral
  • อีเมลธุรกรรมครบชุด — ยินดีต้อนรับ · ใบเสร็จ · แจ้งเตือนหมดอายุ · รีเซ็ตรหัสผ่าน — ทั้งหมด branded HTML
  • หน้า Landing + Signup + Onboarding — พร้อมรองรับลูกค้าใหม่
  • Google Sign-In — เข้าสู่ระบบด้วยบัญชี Google ได้
  • ชื่อร้านในหัวแคชเชียร์ — แสดงชื่อแบรนด์ tenant ถูกต้องทุกหน้า (data-brand-name + server-side rewrite)
  • DB Indexes SaaS — เพิ่ม 6 indexes สำหรับ hot-path multi-tenant query (tenant_id scoping)
  • อีเมล dunning รวมศูนย์ — ลบ HTML generator ซ้ำ 3 ตัว → ใช้ billingHtml ตัวเดียว
  • แจ้งเตือน LINE auto-reply — ลูกค้าทักมา LINE OA ได้รับ reply อัตโนมัติ ชี้ไปสแกน QR
  • Onboarding wizard — Step 2 (รับออเดอร์แรก) ติ๊กถูกอัตโนมัติหลังรับออเดอร์จริงครั้งแรก
  • Plan quota meter — ⚙ ตั้งค่า > แพ็กเกจ แสดง progress bar ออเดอร์ใช้ไปแล้ว/เดือน (Free plan)

🕐 รอขึ้น Production

ขึ้น saas branch สู่ Production รอเจ้าของสั่งโค้ดพร้อมทุกอย่าง — รอ Render upgrade + ตั้งค่า env (Omise · SendGrid · BASE_URL)
ออกแบบหน้าจ่ายเงินลูกค้าใหม่ รอตัดสินใจให้สะอาด/เข้าใจง่ายขึ้น

📋 สิ่งที่เจ้าของต้องเตรียม (เพื่อขึ้นใช้จริงเต็มรูปแบบ)

เครื่องปริ้นใบเสร็จ/สลิปครัว รอฮาร์ดแวร์โค้ดเขียนรอไว้แล้ว เสียบเครื่อง (เช่น Sunmi) แล้วเปิดใช้ได้
พร้อมเพย์แบบร้านค้า (สำหรับตรวจสลิปอัตโนมัติ) รอคีย์K SHOP เดิมใช้กับ SlipOK ไม่ได้ — ตอนนี้ใช้แคชเชียร์กดยืนยันเอง
LINE token (แจ้งเตือนเจ้าของ) รอคีย์สรุปยอดรายวัน + เตือนของใกล้หมดเข้า LINE
ฐานข้อมูลถาวรของ UAT ทางเลือกตอนนี้ UAT รีเซ็ตเองทุกครั้ง — ถ้าอยากให้ข้อมูลทดสอบอยู่ถาวรค่อยตั้ง

💬 คุยกันต่อบนมือถือยังไง

  • เปิด แอป Claude บนมือถือ (claude.ai) — บอกสิ่งที่อยากได้/อยากแก้
  • ส่งลิงก์หน้านี้ หรือก๊อปหัวข้อให้ Claude ดู → ต่อบริบทได้ทันที
  • เข้าดูผลงานจริงที่ลิงก์ด้านบนได้ตลอด (ไม่ต้องผ่านใคร)
  • กลับมาเปิดคอม → resume งานในเครื่องเพื่อแก้โค้ด + ขึ้น UAT ให้
หมายเหตุ: แอป Claude บนมือถือคุย/วางแผนได้ แต่ แก้โค้ดในระบบเองไม่ได้ — การลงมือแก้จริงทำตอนเปิดคอม
