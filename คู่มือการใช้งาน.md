# คู่มือระบบ Mobile Queue 🎫

ระบบจองคิวแบบ **สแกน QR แล้วรับคิวผ่าน LINE** ลูกค้าไม่ต้องโหลดแอป ไม่ต้องสมัครสมาชิก
สแกน QR หน้าร้าน → รับหมายเลขคิวทาง LINE → ระบบแจ้งเตือนเมื่อใกล้ถึงคิว และเมื่อถึงคิว

เอกสารนี้แบ่งเป็น 2 ส่วน:
- **ส่วนที่ 1 — สำหรับช่างเทคนิค / ผู้ติดตั้ง** (ติดตั้งระบบ ตั้งค่า LINE นำขึ้นเซิร์ฟเวอร์)
- **ส่วนที่ 2 — สำหรับผู้ใช้งาน** (พนักงานหน้าร้าน และลูกค้า)

---

# ส่วนที่ 1 — คู่มือช่างเทคนิค / ผู้ติดตั้ง

## 1.1 สิ่งที่ต้องเตรียม
- เครื่องคอมพิวเตอร์ที่ติดตั้ง **Node.js เวอร์ชัน 22.5 ขึ้นไป** ([ดาวน์โหลด](https://nodejs.org))
- บัญชี **LINE Official Account** (สมัครฟรี) — สำหรับส่งแจ้งเตือนจริง
- โดเมนที่เปิดผ่าน **HTTPS** (จำเป็นสำหรับ LINE) เช่นโฮสต์บน Railway, Render, Fly.io หรือ VPS

> ℹ️ ถ้ายังไม่ตั้งค่า LINE ระบบจะ "จำลอง" การส่ง โดยพิมพ์ข้อความลงหน้าจอเซิร์ฟเวอร์แทน
> ทำให้ทดสอบทั้งระบบได้ก่อนที่จะเชื่อม LINE จริง

## 1.2 ติดตั้งและทดสอบบนเครื่อง (โหมดทดสอบ ยังไม่ต้องมี LINE)

เปิด Terminal / Command Prompt ในโฟลเดอร์ `mobile-queue` แล้วรันทีละคำสั่ง:

```bash
npm install            # ติดตั้งไลบรารีที่ใช้
copy .env.example .env   # (Windows) หรือ  cp .env.example .env  (Mac/Linux)
npm run seed           # สร้างร้านตัวอย่าง + 3 โซน
npm run qr             # สร้างไฟล์ QR ไว้ที่ public/assets/qr/
npm start              # เปิดเซิร์ฟเวอร์ที่ http://localhost:3000
```

เปิดเบราว์เซอร์ทดสอบ:
- หน้าพนักงาน: `http://localhost:3000/cashier/` (PIN เริ่มต้น = `1234`)
- หน้าจอแสดงคิว: `http://localhost:3000/display/`
- หน้าลูกค้า: `http://localhost:3000/liff/?zone=1`

ลองกด **เรียกคิวถัดไป** ที่หน้าพนักงาน แล้วดูหน้าจอแสดงคิวและหน้าลูกค้าอัปเดตทันที
(ในโหมดทดสอบ ข้อความ LINE จะปรากฏที่หน้าจอเซิร์ฟเวอร์ ไม่ได้ส่งจริง)

> ⚠️ จะมีข้อความเตือน "experimental feature (SQLite)" หนึ่งบรรทัด — เป็นเรื่องปกติ ไม่ใช่ข้อผิดพลาด

## 1.3 ตั้งค่า LINE เพื่อส่งแจ้งเตือนจริง

### ขั้นที่ 1 — สร้าง Messaging API Channel
1. เข้า [LINE Developers Console](https://developers.line.biz/console/)
2. สร้าง **Provider** แล้วสร้าง **Messaging API channel** (จะได้ Official Account มาด้วย)
3. ในแท็บ **Messaging API**:
   - ออก **Channel access token (long-lived)** → นำไปใส่ `LINE_CHANNEL_ACCESS_TOKEN`
   - คัดลอก **Channel secret** (แท็บ Basic settings) → ใส่ `LINE_CHANNEL_SECRET`
   - เปิด **Use webhook** และตั้ง Webhook URL = `https://โดเมนของคุณ/line/webhook`
   - ปิด "Auto-reply messages" และ "Greeting messages" ถ้าต้องการให้ระบบตอบเอง

### ขั้นที่ 2 — สร้าง LIFF App (หน้าลูกค้าแบบไม่ต้องสมัคร)
1. ใน channel เดิม เปิดแท็บ **LIFF** → กด **Add**
2. ตั้งค่า:
   - **Endpoint URL**: `https://โดเมนของคุณ/liff/`
   - **Size**: Full
   - **Scopes**: เลือก `profile` และ `openid`
3. คัดลอก **LIFF ID** → ใส่ `LIFF_ID`

> หน้าลูกค้าจะดึง LINE userId อัตโนมัติผ่าน LIFF จึงไม่ต้องกรอกเบอร์โทรหรือสมัครสมาชิก

### ขั้นที่ 3 — แก้ไขไฟล์ `.env`
```ini
PORT=3000
PUBLIC_BASE_URL=https://โดเมนของคุณ
CASHIER_PIN=ตั้งรหัสเอง
NOTIFY_THRESHOLD=2            # แจ้งเตือนเมื่อเหลืออีกกี่คิวก่อนถึง
LINE_CHANNEL_ACCESS_TOKEN=...
LINE_CHANNEL_SECRET=...
LIFF_ID=...
```

### ขั้นที่ 4 — สร้าง QR ใหม่ให้ชี้ไปที่ LINE
```bash
npm run qr
```
เมื่อใส่ `LIFF_ID` แล้ว QR แต่ละโซนจะชี้ไปที่ `https://liff.line.me/<LIFF_ID>?zone=<id>`
สแกนแล้วเปิดในแอป LINE ทันที — พิมพ์ไฟล์ `public/assets/qr/zone-<id>.png` ไปติดตามจุด/โซนที่ตรงกัน

### ขั้นที่ 5 — นำขึ้นเซิร์ฟเวอร์ (Deploy)
- ใช้โฮสต์ที่รัน Node 22+ และเปิด HTTPS ได้ (Railway, Render, Fly.io, VPS + Caddy/Nginx)
- LINE บังคับใช้ **HTTPS** ทั้ง Webhook และ LIFF
- ถ้าจะทดสอบ LINE จริงจากเครื่องตัวเอง ใช้ tunnel เช่น `ngrok http 3000` หรือ `cloudflared`
  แล้วเอา URL HTTPS ที่ได้ไปใส่เป็น `PUBLIC_BASE_URL` / Webhook / LIFF Endpoint

## 1.4 ตั้งค่าร้านและโซนของจริง
ตอนนี้ใช้ข้อมูลตัวอย่าง (ร้าน "Demo Shabu House" 3 โซน) จากคำสั่ง `npm run seed`
ถ้าต้องการแก้ชื่อร้าน/โซน/อักษรนำหน้าคิว ให้แก้ในไฟล์ `scripts/seed.js`
แล้วลบไฟล์ `data/queue.db` เพื่อเริ่มใหม่ จากนั้นรัน `npm run seed` อีกครั้ง

| สิ่งที่ตั้งค่าได้ | ที่ไฟล์ |
|---|---|
| ชื่อร้าน, โซน, อักษรนำหน้า (A/B/C) | `scripts/seed.js` |
| จำนวนคิวก่อนแจ้งเตือน (NOTIFY_THRESHOLD) | `.env` |
| รหัส PIN พนักงาน | `.env` (`CASHIER_PIN`) |

## 1.5 การดูแล / ข้อจำกัด
- ฐานข้อมูลเก็บที่ไฟล์ `data/queue.db` — **ควรสำรอง (backup) ไฟล์นี้สม่ำเสมอ**
- PIN เป็นการป้องกันแบบเบาๆ เหมาะกับแท็บเล็ตหน้าร้าน ไม่ควรเปิดหน้าพนักงานสู่อินเทอร์เน็ตสาธารณะ
  ควรมีระบบเครือข่าย/รหัสผ่านของร้านครอบอีกชั้น
- ระบบนี้เหมาะกับ 1 เซิร์ฟเวอร์ (ร้านเดียวหรือเชนเล็ก) ถ้าจะขยายหลายสาขาพร้อมกันจำนวนมาก
  ควรเปลี่ยนไปใช้ Postgres + Redis (แก้เฉพาะไฟล์ `server/db.js` และ `server/events.js`)
- เสียงเรียกคิวบนหน้าจอใช้เสียงสังเคราะห์ของเบราว์เซอร์ เสียงภาษาไทยขึ้นกับระบบปฏิบัติการ
  ต้องกดปุ่ม **🔊 เปิดเสียง** หนึ่งครั้งก่อน (เบราว์เซอร์บล็อกเสียงอัตโนมัติจนกว่าจะมีการคลิก)

# ส่วนที่ 2 — คู่มือผู้ใช้งาน

## 2.1 สำหรับลูกค้า 📱

1. **สแกน QR** ที่ติดอยู่หน้าร้าน/โซนที่ต้องการ (สแกนด้วยกล้องหรือ LINE)
2. หน้าจอจะแสดงชื่อร้านและโซน — **เลือกจำนวนคน** ด้วยปุ่ม `–` / `+`
3. กดปุ่ม **รับคิว** → ระบบให้หมายเลขคิว (เช่น `A005`)
4. รอได้ตามสบาย ระบบจะส่ง LINE แจ้งเตือนให้ 2 ครั้ง:
   - ⏰ **ใกล้ถึงคิวแล้ว** — เมื่อเหลืออีกไม่กี่คิว (ให้เดินกลับมาที่ร้าน)
   - 🔔 **ถึงคิวของคุณแล้ว** — ให้ไปที่เคาน์เตอร์
5. ดูสถานะคิวปัจจุบันได้ตลอดในหน้าเดิม (เหลืออีกกี่คิว / เรียกถึงเลขอะไรแล้ว)
6. ถ้าไม่ต้องการต่อแล้ว กด **ยกเลิกคิว** ได้

> ลูกค้าไม่ต้องโหลดแอปหรือสมัครสมาชิกใดๆ ใช้ผ่าน LINE ได้เลย

## 2.2 สำหรับพนักงานหน้าร้าน (หน้า Cashier) 🧑‍💼

เปิดหน้า `/cashier/` บนแท็บเล็ตหรือคอมพิวเตอร์ แล้วใส่ **PIN**

| ปุ่ม / ส่วน | หน้าที่ |
|---|---|
| เลือกโซน (มุมขวาบน) | สลับดูคิวของแต่ละโซน |
| **📢 เรียกคิวถัดไป** | เรียกคิวหมายเลขถัดไป (ระบบส่ง LINE หาลูกค้าคนนั้นว่า "ถึงคิวแล้ว") |
| **เสร็จ / Served** | ปิดคิวที่เรียกแล้วเมื่อลูกค้ามารับบริการ |
| **ข้าม / Skip** | ข้ามคิวที่ลูกค้าไม่มาแสดงตัว |
| **ปิดโซน / เปิดโซน** | เมื่อคิวเต็มหรือปิดให้บริการโซนนั้น (ลูกค้าจะรับคิวใหม่ไม่ได้) |

- รายการ **กำลังรอ** แสดงคิวที่ยังไม่ได้เรียก (ป้าย "แจ้งแล้ว" = ส่งเตือนใกล้ถึงคิวไปแล้ว)
- รายการ **เรียกแล้ว** แสดงคิวที่เพิ่งเรียก พร้อมปุ่มกดว่าให้บริการเสร็จ
- หน้าจ้ออัปเดตอัตโนมัติแบบเรียลไทม์ ไม่ต้องรีเฟรช

## 2.3 หน้าจอแสดงคิว (หน้า Display) 🖥️

เปิด `/display/` บนทีวี/จอในร้าน

- ฝั่งซ้ายตัวใหญ่ = **คิวที่กำลังเรียก (NOW SERVING)**
- ฝั่งขวา = **คิวถัดไป (NEXT UP)**
- ด้านล่าง = คิวที่เรียกล่าสุด
- กดปุ่ม **🔊 เปิดเสียง** หนึ่งครั้ง เพื่อให้มีเสียงประกาศเรียกคิว เช่น "ขอเชิญหมายเลข A 0 0 1 ค่ะ"
- เลือกโซนที่ต้องการแสดงได้จากมุมขวาบน

## สรุปการทำงานของระบบคิว
- แต่ละโซนมีหมายเลขของตัวเอง พร้อมอักษรนำหน้า เช่น `A001`, `B001`
- ลูกค้าสแกน → เลือกจำนวนคน → รับคิว → ได้ข้อความยืนยันทาง LINE
- พนักงานกด "เรียกคิวถัดไป" → ลูกค้าคนนั้นได้ข้อความ "ถึงคิวแล้ว"
- ทุกครั้งที่เรียก/ปิดคิว ระบบจะคำนวณใหม่ ใครที่เหลือไม่เกินค่าที่ตั้งไว้ (เช่น 2 คิว)
  และยังไม่เคยถูกเตือน จะได้ข้อความ "ใกล้ถึงคิว" **หนึ่งครั้ง**

ตัวอย่าง (ตั้งค่าแจ้งเตือน = 2):
```
คิว: A001 A002 A003 A004 A005
เรียก A001  →  A002, A003, A004 ได้ "ใกล้ถึงคิว"   (A005 เหลือ 3 คิว ยังไม่เตือน)
เรียก A002  →  A005 เหลือ 2 คิว → ได้ "ใกล้ถึงคิว"
```

## ติดต่อ / แก้ปัญหาเบื้องต้น
| อาการ | วิธีแก้ |
|---|---|
| เปิดเซิร์ฟเวอร์ไม่ขึ้น | ตรวจ Node.js ต้องเป็นเวอร์ชัน 22.5+ (`node -v`) |
| QR สแกนแล้วไม่เปิด LINE | ตรวจว่าตั้ง `LIFF_ID` แล้วรัน `npm run qr` ใหม่ |
| LINE ไม่ส่งข้อความ | ตรวจ `LINE_CHANNEL_ACCESS_TOKEN`, `LINE_CHANNEL_SECRET` และ Webhook URL เป็น HTTPS |
| หน้าจอไม่มีเสียง | กดปุ่ม 🔊 เปิดเสียง หนึ่งครั้งก่อน |
| พนักงานกดเรียกคิวไม่ได้ (PIN ไม่ถูกต้อง) | ตรวจ PIN ให้ตรงกับ `CASHIER_PIN` ใน `.env` |
